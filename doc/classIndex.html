Class Hierarchy

  • fabula.Asset - A single asset entry.
  • fabula.assets.Assets - An assets manager which returns file-like objects for local files. It is used to retrieve media data - images, animations, 3D models, sound.
  • fabula.Event - Fabula event base class.
    • fabula.AttemptEvent - This is the base class for attempt events by the player or NPCs.
      • fabula.TriesToDropEvent - The player or NPC tries to drop an item.
      • fabula.TriesToLookAtEvent - Issued when the player or an NPC looks at a map element or an item.
      • fabula.TriesToManipulateEvent - Issued when the player or an NPC tries to manipulate an item. NPCs cannot be manipulated.
      • fabula.TriesToMoveEvent - Issued when the player or an NPC wants to move, awaiting server confirmation.
      • fabula.TriesToPickUpEvent - Issued when the player or an NPC tries to pick up an item. Attempts to pick up NPCs are discarded or answered with an error.
      • fabula.TriesToTalkToEvent - Issued when the player or an NPC wants to start a conversation with another entity. It is not possible to talk to items.
    • fabula.ChangePropertyEvent - This Event is issued to trigger a property change. Each instance of fabula.Entity has a property.
    • fabula.ConfirmEvent - This is the base class for server confirmations of attempt events.
      • fabula.AttemptFailedEvent - No summary
      • fabula.CanSpeakEvent - This is a server invitation for the player or an NPC to speak. Since this event requires immediate user input, it is always executed as the last event of a Message by the UserInterface.
      • fabula.DropsEvent - No summary
      • fabula.ManipulatesEvent - No summary
      • fabula.MovesToEvent - This is the server confirmation of a movement.
      • fabula.PerceptionEvent - No summary
      • fabula.PicksUpEvent - This is a server confirmation of a TriesToPickUpEvent issued by the player or an NPC.
    • fabula.ExitEvent - This Event is sent by the Client or the Server to signal that it is about to exit and that no further Events will be sent.
    • fabula.InitEvent - No summary
    • fabula.PassiveEvent - The base class for events when items or NPCs are being passed etc.
      • fabula.DroppedEvent - Issued by the server when an item is being dropped by the player or an NPC.
      • fabula.LookedAtEvent - Issued by the server when an item or NPC is being looked at by the player or an NPC.
      • fabula.PassedEvent - Issued by the server when an item or an NPC is being passed by the player or an NPC.
      • fabula.PickedUpEvent - Issued by the server when an item is being picked up by the player or an NPC.
    • fabula.SaysEvent - No summary
    • fabula.ServerEvent - Base class for various events issued by the server that are no ConfirmEvents, including map and room events.
      • fabula.ChangeMapElementEvent - This event changes a single element of the two-dimensional map. When sent in a Message, these events are executed immediately and in parallel upon the rendering of the Message by the UserInterface.
      • fabula.DeleteEvent - This event deletes an item or NPC from the map. Note that items may still persist in the players posessions.
      • fabula.EnterRoomEvent - No summary
      • fabula.RoomCompleteEvent - Issued by the server after an EnterRoomEvent when the room is populated with a tiles, a player Entity, items and NPCs.
      • fabula.ServerParametersEvent - This Event is sent by the Server when an InitEvent has been received. It informs the client about the Server parameters.
      • fabula.SpawnEvent - This event creates the player / new item / NPC on the map.
  • fabula.eventprocessor.EventProcessor - This is the base class for all Fabula objects that process events.
    • fabula.core.Engine - Common base class for Fabula server and client engines. Most likely you will want to override all of the methods when subclassing Engine, so the use of this class is providing a structure for Engines.
      • fabula.core.client.Client - No summary
      • fabula.core.server.Server - The Server is the central management and control authority in Fabula. It relies on the ServerInterface and the Plugin.
    • fabula.Entity - An Entity is a Fabula game object. This is the base class for the player, NPCs and items. An Entity holds information used by the Fabula game logic:
      • fabula.plugins.pygameui.PygameEntity - Pygame-aware subclass of Entity to be used in PygameUserInterface.
    • fabula.plugins.Plugin - Base class for plugins to be used by a fabula.core.Engine.
      • fabula.plugins.serverside.DefaultGame - This is an off-the-shelf server plugin, running a default Fabula game.
        • fabula.plugins.serverside.CommandLine - This is a command-line interface server plugin.
        • fabula.plugins.serverside.Editor - This is the server-side Plugin for a Fabula map editor.
      • fabula.plugins.ui.UserInterface - No summary
        • fabula.plugins.pygameui.PygameUserInterface - This is a Pygame implementation of an UserInterface for the Fabula Client.
          • fabula.plugins.pygameui.PygameEditor - A Pygame-based user interface that serves as a map editor.
    • fabula.Room - A Room instance saves the map as well as Entities and their locations. It is maintained for the current room in the Client and for all active rooms in the Server.
  • fabula.FloorPlanElement - Convenience class for floor plan elements.
  • fabula.interfaces.Interface - This is a base class for Fabula interfaces which handle all the network traffic. A customised implementation will likely have a client- and a server side interface.
    • fabula.interfaces.python_tcp.TCPClientInterface - Fabula Client interface using TCP.
    • fabula.interfaces.python_tcp.TCPServerInterface - Fabula Server interface using TCP.
      • fabula.interfaces.json_rpc.JSONRPCServerInterface - Fabula Server interface using JSON-RPC.
    • fabula.interfaces.replay.PythonReplayInterface - An Interface which replays Python message logs.
    • fabula.interfaces.StandaloneInterface - An Interface that is meant to be used in conjunction with run.App.run_standalone().
  • fabula.interfaces.MessageBuffer - Buffer messages received and to be sent over the network.
  • fabula.Message - A Message represents a set of Events that happen in parallel.
  • fabula.plugins.pygameui.PygameOSD - A PygameOSD instance manages an on-screen display.
  • fabula.Rack - A Rack stores Entities removed from a room. An instance of Rack is used by each Engine.
  • fabula.run.App - An App instance represents a Fabula client or server application.
  • fabula.Tile - A Tile is an element of a two-dimensional map. It is never meant to perform any active logic, so its only property is a type and an asset.
  • Interface
    • fabula.interfaces.python_udp.UDPClientInterface - This is the base class for a Fabula Client Interface. Implementations should subclass this one an override the default methods, which showcase a simple example.
    • fabula.interfaces.python_udp.UDPServerInterface - This is the base class for a Fabula Server Interface. Implementations should subclass this one an override the default methods, which do nothing.
    • fabula.interfaces.twisted_tcp.TCPInterface - A generic Fabula Interface using TCP, built upon the Twisted framework.
  • MessageBuffer
    • fabula.interfaces.python_udp.UDPClientInterface - This is the base class for a Fabula Client Interface. Implementations should subclass this one an override the default methods, which showcase a simple example.
    • fabula.interfaces.twisted_tcp.ProtocolMessageBuffer - Buffer messages received and to be sent over the network. An instance of this class is created by a Twisted Factory every time a connection is made.
  • planes.gui.Container
    • fabula.plugins.pygameui.EventEditor - A planes Container that implements an editor for Fabula Events.
  • planes.Plane
    • fabula.plugins.pygameui.EntityPlane - Subclass of Plane with an extended update() method that handles movement.
  • twisted.internet.protocol.Protocol
    • fabula.interfaces.twisted_tcp.ProtocolMessageBuffer - Buffer messages received and to be sent over the network. An instance of this class is created by a Twisted Factory every time a connection is made.
